# Collecta

Loan collection management system. Next.js web app, Hono API, shared TypeScript types, Postgres with Drizzle.

## Requirements

- Bun or npm
- Docker and Docker Compose
- OpenSSL

## Project Structure

```text
apps/web          Next.js web app
apps/api          Hono API server
packages/db       Drizzle schema, database client, and seed data
packages/shared   Shared TypeScript types
infra             Docker Compose, Dockerfile, and Caddy config
scripts           Setup, clean, and local environment scripts
```

## Quick Start

```bash
bun run setup       # or: npm run setup
```

This creates `.env` (if missing), installs dependencies, starts Docker services, pushes the DB schema, and seeds demo data.

The setup script auto-detects Bun when available and falls back to npm. Force npm:

```bash
FORCE_NPM=1 bun run setup
npm run setup -- --npm
```

After setup:

| Service | URL |
| --- | --- |
| Web app | `http://localhost:3000` |
| Caddy proxy | `http://localhost` |
| API health | `http://localhost:4000/api/v1/health` |

## Environment Variables

Generated by `setup:env` with `openssl`. Printed to stdout on creation. Does not overwrite an existing `.env`.

```bash
bun run setup:env                  # or: npm run setup:env
SETUP_ENV_FORCE=1 bun run setup:env  # force regenerate
ENV_FILE=.env.local SETUP_ENV_FORCE=1 bun run setup:env  # write elsewhere
```

| Variable | Default | Required | Used by |
| --- | --- | --- | --- |
| `NODE_ENV` | `development` | No | API, Docker |
| `PORT` | `4000` | No | API, Docker |
| `JWT_SECRET` | generated (openssl 64B) | Yes | API, web proxy |
| `JWT_EXPIRES_IN` | `8h` | No | API |
| `AUTH_MAX_FAILED_LOGIN_ATTEMPTS` | `5` | No | API |
| `AUTH_LOCK_DURATION_SECONDS` | `900` | No | API |
| `POSTGRES_USER` | `collecta` | Yes | API, Drizzle, Docker |
| `POSTGRES_PASSWORD` | generated (openssl 24B) | Yes | API, Drizzle, Docker |
| `POSTGRES_DB` | `collecta` | Yes | API, Drizzle, Docker |
| `POSTGRES_HOST` | `localhost` | Yes | API, Drizzle (Docker overrides to `postgres`) |
| `POSTGRES_PORT` | `5432` | Yes | API, Drizzle, Docker |
| `NEXT_PUBLIC_SHOW_DEMO_ACCOUNTS` | `true` | No | Web |
| `NEXT_PUBLIC_API_URL` | `/api/v1` | No | Web |

`.env` is Git-ignored.

## Docker Services

`infra/docker-compose.yml` starts four containers:

| Service | Image | Port | Notes |
| --- | --- | --- | --- |
| `collecta-web` | `collecta-dev:latest` | `3000` | Next.js dev server |
| `collecta-api` | `collecta-dev:latest` | `4000` | Hono API |
| `collecta-postgres` | `postgres:17-alpine` | `5432` | Database |
| `collecta-caddy` | `caddy:2-alpine` | `80` | Reverse proxy |

Caddy routes `/api/*` → API container, everything else → web container.

The Docker image always uses **Bun** internally. Anonymous volumes protect the image's `node_modules` from being overridden by host mounts, so the containers work regardless of whether the host used Bun or npm to install dependencies.

## Common Workflows

```bash
# reset database
bun run clean:db && bun run up && bun run db:push && bun run db:seed

# regenerate deps
bun run clean:web && bun run clean:api && bun install

# full destructive reset
bun run clean && bun run setup
```

## Troubleshooting

**Mobile Safari hydration fails** — Restart the web dev server after config changes. `apps/web/next.config.ts` allows LAN origins via `allowedDevOrigins`.

**Typecheck errors in `.next/dev/types`** — Remove the dev cache and restart:

```bash
bun run clean:web
```

**Docker containers missing modules after `npm run setup`** — Anonymous volumes preserve the image's `node_modules`. Rebuild if needed:

```bash
docker compose --env-file .env -f infra/docker-compose.yml build
bun run up
```

**`npm install` fails with `workspace:*`** — The setup script rewrites `workspace:*` to `file:` temporarily for npm and restores originals afterward. Run `setup` rather than `npm install` directly.

## Quick Start

Run the full setup:

```bash
bun run setup
```

If Bun is not installed, run the script through npm instead:

```bash
npm run setup
```

The setup script uses Bun when it is available and falls back to npm when Bun is not in `PATH`. To force npm even when Bun is installed:

This will:

- Wait for Postgres
- Push the Drizzle schema with `bun run db:push`

After setup, open:

- Web app: `http://localhost:3000`
- Caddy reverse proxy: `http://localhost`

Force regeneration:

## Environment Variables

| Variable | Default from setup | Required | Used by | Description |
| --- | --- | --- | --- | --- |
| `NODE_ENV` | `development` | No | API, Docker | Runtime environment. Controls secure cookie behavior in the API. |
| `PORT` | `4000` | No | API, Docker | API server port inside and outside Docker. |
| `JWT_SECRET` | Generated by OpenSSL | Yes | API, web proxy | Secret used to sign and verify session JWTs. Keep private. |
| `JWT_EXPIRES_IN` | `8h` | No | API | Session JWT lifetime. |
| `AUTH_MAX_FAILED_LOGIN_ATTEMPTS` | `5` | No | API | Failed login attempts before account lock. |
| `AUTH_LOCK_DURATION_SECONDS` | `900` | No | API | Account lock duration after too many failed attempts. |
| `POSTGRES_USER` | `collecta` | Yes | API, Drizzle, Docker | Postgres username. |
| `POSTGRES_PASSWORD` | Generated by OpenSSL | Yes | API, Drizzle, Docker | Postgres password. Keep private. |
| `POSTGRES_DB` | `collecta` | Yes | API, Drizzle, Docker | Postgres database name. |
| `POSTGRES_HOST` | `localhost` | Yes | API, Drizzle | Host used by local commands. Docker overrides the API container to use `postgres`. |
| `POSTGRES_PORT` | `5432` | Yes | API, Drizzle, Docker | Postgres port exposed to the host. |
| `NEXT_PUBLIC_SHOW_DEMO_ACCOUNTS` | `true` | No | Web | Shows seeded demo account shortcuts on `/login`. |
| `NEXT_PUBLIC_API_URL` | `/api/v1` | No | Web | Browser API base URL. Keep `/api/v1` when using Caddy/Next routing. |

`.env` is ignored by Git.

## Troubleshooting

If mobile Safari cannot hydrate the dev app over LAN, make sure the web dev server has been restarted after config changes. `apps/web/next.config.ts` allows common LAN origins for Next dev resources.

