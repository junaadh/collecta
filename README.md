# Collecta

Collecta is a loan collection management system with a Next.js web app, Hono API, shared TypeScript types, and a Postgres database managed with Drizzle.

## Requirements

- Bun
- Docker and Docker Compose
- OpenSSL

## Project Structure

```text
apps/web          Next.js web app
apps/api          Hono API server
packages/db       Drizzle schema, database client, and seed data
packages/shared   Shared TypeScript types
infra             Docker Compose, Dockerfile, and Caddy config
scripts           Setup, clean, and local environment scripts
```

## Quick Start

Run the full setup:

```bash
bun run setup
```

This will:

- Create `.env` if it does not already exist
- Print the generated `.env` values to stdout
- Run `bun install`
- Start Docker services with `bun run up`
- Wait for Postgres
- Push the Drizzle schema with `bun run db:push`
- Seed demo data with `bun run db:seed`

After setup, open:

- Web app: `http://localhost:3000`
- Caddy reverse proxy: `http://localhost`
- API health check: `http://localhost:4000/api/v1/health`

## Environment Setup

Generate only the `.env` file:

```bash
bun run setup:env
```

`setup:env` uses `openssl` to generate secrets and prints the resulting file to stdout. If `.env` already exists, it keeps the existing file.

Force regeneration:

```bash
SETUP_ENV_FORCE=1 bun run setup:env
```

Write to a different file:

```bash
ENV_FILE=.env.local SETUP_ENV_FORCE=1 bun run setup:env
```

## Environment Variables

| Variable | Default from setup | Required | Used by | Description |
| --- | --- | --- | --- | --- |
| `NODE_ENV` | `development` | No | API, Docker | Runtime environment. Controls secure cookie behavior in the API. |
| `PORT` | `4000` | No | API, Docker | API server port inside and outside Docker. |
| `JWT_SECRET` | Generated by OpenSSL | Yes | API, web proxy | Secret used to sign and verify session JWTs. Keep private. |
| `JWT_EXPIRES_IN` | `8h` | No | API | Session JWT lifetime. |
| `AUTH_MAX_FAILED_LOGIN_ATTEMPTS` | `5` | No | API | Failed login attempts before account lock. |
| `AUTH_LOCK_DURATION_SECONDS` | `900` | No | API | Account lock duration after too many failed attempts. |
| `POSTGRES_USER` | `collecta` | Yes | API, Drizzle, Docker | Postgres username. |
| `POSTGRES_PASSWORD` | Generated by OpenSSL | Yes | API, Drizzle, Docker | Postgres password. Keep private. |
| `POSTGRES_DB` | `collecta` | Yes | API, Drizzle, Docker | Postgres database name. |
| `POSTGRES_HOST` | `localhost` | Yes | API, Drizzle | Host used by local commands. Docker overrides the API container to use `postgres`. |
| `POSTGRES_PORT` | `5432` | Yes | API, Drizzle, Docker | Postgres port exposed to the host. |
| `NEXT_PUBLIC_SHOW_DEMO_ACCOUNTS` | `true` | No | Web | Shows seeded demo account shortcuts on `/login`. |
| `NEXT_PUBLIC_API_URL` | `/api/v1` | No | Web | Browser API base URL. Keep `/api/v1` when using Caddy/Next routing. |

`.env` is ignored by Git.

## Scripts

### Setup

| Command | Description |
| --- | --- |
| `bun run setup:env` | Generate `.env` only and print values to stdout. Does not install dependencies or start services. |
| `bun run setup` | Full local setup: create `.env` if missing, install dependencies, start Docker, push schema, and seed data. |

### Development

| Command | Description |
| --- | --- |
| `bun run dev` | Start API and web dev servers together. |
| `bun run dev:web` | Start the Next.js web app on port `3000`. |
| `bun run dev:api` | Start the Hono API on port `4000`. |
| `bun run up` | Start Docker services from `infra/docker-compose.yml`. |
| `bun run down` | Stop Docker services without deleting volumes. |

### Database

| Command | Description |
| --- | --- |
| `bun run db:generate` | Generate Drizzle migrations from schema changes. |
| `bun run db:migrate` | Run generated Drizzle migrations. |
| `bun run db:push` | Push the current schema directly to Postgres. Useful for local development. |
| `bun run db:studio` | Open Drizzle Studio. |
| `bun run db:seed` | Reset and seed demo data. |

### Quality

| Command | Description |
| --- | --- |
| `bun run lint` | Run ESLint for the web app. |
| `bun run typecheck` | Typecheck API and web. |
| `bun run typecheck:api` | Typecheck only the API. |
| `bun run typecheck:web` | Typecheck only the web app. |

### Cleaning

| Command | Description |
| --- | --- |
| `bun run clean` | Remove local web/API/package build caches, all workspace `node_modules`, and Bun package cache. Keeps Docker and database data. |
| `bun run clean:web` | Remove Next/web build caches and `apps/web/node_modules/.cache`. |
| `bun run clean:api` | Remove API/Hono build caches and `apps/api/node_modules/.cache`. |
| `bun run clean:db` | Stop Compose services and drop the Postgres Docker volume. This deletes local database data. |
| `bun run clean:docker` | Stop Compose services and prune Docker containers, images, volumes, and build cache. This is destructive. |

Missing cache paths are ignored. Each clean target prints a final success message.

## Docker Services

`infra/docker-compose.yml` starts:

- `web`: Next.js dev server on `3000`
- `api`: Hono API on `4000`
- `postgres`: Postgres on `5432`
- `caddy`: reverse proxy on `80`

Caddy proxies `/api/*` to the API container and all other traffic to the web container.

## Common Workflows

Reset the local database:

```bash
bun run clean:db
bun run up
bun run db:push
bun run db:seed
```

Regenerate dependencies from scratch:

```bash
bun run clean:web
bun run clean:api
bun install
```

Full destructive reset:

```bash
bun run clean
bun run setup
```

## Troubleshooting

If mobile Safari cannot hydrate the dev app over LAN, make sure the web dev server has been restarted after config changes. `apps/web/next.config.ts` allows common LAN origins for Next dev resources.

If `bun run typecheck:web` reports errors in `.next/dev/types`, remove the generated Next dev cache:

```bash
bun run clean:web
```

Then restart the web dev server.
